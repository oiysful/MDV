# 06. 에디터에 마크다운 보조가 전혀 없음

## 문제

소스 에디터의 키 처리는 **Tab 하나가 전부다.**

```js
// src/renderer/editor.js:236-243
editor.addEventListener('keydown', event => {
  if (event.key === 'Tab') {
    event.preventDefault()
    if (!document.execCommand('insertText', false, '\t')) { ... }
  }
})
```

없는 것:

- **리스트 이어쓰기** — `- item`에서 Enter를 쳐도 다음 줄에 `- `가 안 붙는다. 번호 목록, 체크박스도 마찬가지. 빈 리스트 항목에서 Enter로 리스트를 빠져나오는 동작도 없다.
- **⌘B / ⌘I** — 굵게·기울임 단축키가 없다. 선택 영역을 `**`/`*`로 감싸는 기본 동작.
- **워드랩** — `white-space: pre` (`src/renderer/index.html:632`). 긴 문단이 가로로 무한히 스크롤된다. 분할 모드에서 pane이 좁아지면(`minmax(300px,1fr)`) 특히 괴롭다.
- 인용문(`>`) 이어쓰기, 자동 괄호/따옴표 짝 맞춤도 없음.

## 왜 중요한가

여기가 MDV가 단순히 거친 게 아니라 **기준 미달인** 유일한 지점이다. Typora·Obsidian·iA Writer 전부 Enter로 불릿을 이어주고 ⌘B로 굵게 한다. 현재 MDV의 소스 모드는 사실상 **`<textarea>` 하나**다.

## 접근

전부 `editor.js#bindEditorEvents`(236행 부근) 한 곳에서 처리된다. 새 의존성 없음.

**핵심 원시요소는 이미 있다:** `document.execCommand('insertText', ...)`. 최근 커밋(`c873483`)이 Tab 처리에 이걸 쓴 이유가 바로 **네이티브 undo 스택을 보존하기 위해서**다. 모든 편집 보조를 이 위에 쌓아야 한다. `editor.value = ...`로 직접 대입하면 **undo가 깨진다** — 이게 이 작업의 가장 중요한 제약이다.

### 6a. 리스트 이어쓰기 (Enter)

1. 커서가 속한 현재 줄을 추출.
2. 리스트 접두사 정규식으로 매칭:
   ```js
   /^(\s*)([-*+]|\d+[.)])(\s+\[[ xX]\])?\s+/
   ```
3. **접두사만 있고 내용이 비어 있으면** → 그 접두사를 지운다(리스트 탈출). 이게 없으면 리스트를 빠져나올 방법이 없어서 더 짜증난다.
4. 내용이 있으면 → `\n` + 접두사를 삽입. 번호 목록은 숫자를 +1. 체크박스는 `[ ]`로(체크 상태는 이어받지 않음).
5. 전부 `execCommand('insertText', ...)`로.

### 6b. ⌘B / ⌘I

선택 영역을 `**`/`*`로 감싼다. **토글**로 동작해야 한다 — 이미 감싸여 있으면 벗긴다. 선택이 없으면 마커만 넣고 커서를 가운데로.

**주의:** 전역 keydown 핸들러(`app-runtime.js:336-349`)와 충돌하지 않는지 확인. 현재 ⌘B/⌘I는 잡히지 않으므로 자유롭다. 다만 에디터에 포커스가 있을 때만 동작하도록 에디터 자신의 리스너에서 처리할 것.

### 6c. 워드랩 — ⚠️ 함정 있음

`white-space: pre` → `pre-wrap` 한 줄이면 될 것 같지만 **아니다.**

**줄 번호 거터가 깨진다.** 줄 번호는 `\n` 개수로 생성된다:

```js
// src/renderer/editor.js:2-5
function buildLineNumberText(text) {
  const count = String(text ?? '').split('\n').length
  return Array.from({ length: count }, (_, index) => index + 1).join('\n')
}
```

논리적 줄 1개당 번호 1개다. 소프트랩을 켜면 긴 논리 줄이 **시각적으로 여러 행**을 차지하는데 번호는 여전히 1개 → **거터와 본문이 어긋나기 시작하고, 아래로 갈수록 어긋남이 누적된다.** `#line-highlight`(`index.html:635`)도 같은 이유로 엉뚱한 줄을 강조한다.

(이 함정은 초기 조사에서 "줄 번호는 `\n`을 세므로 그대로 맞다"고 잘못 결론 났던 부분이다. 실제 렌더링을 따져보면 틀렸다.)

**선택지:**

- **(a) 랩 모드에서는 거터를 숨긴다.** 가장 단순하고 정직하다. 랩 토글 시 `#line-numbers`를 `display:none`. 대부분의 마크다운 에디터(iA Writer 등)는 애초에 줄 번호가 없다.
- **(b) 시각적 행 수를 측정해 번호를 정렬한다.** 각 논리 줄의 래핑된 높이를 재서 번호 사이에 여백을 준다. 정확하지만 리사이즈·폰트 변경마다 재계산이 필요하고 성능 부담이 있다.

**(a)를 권한다.** 랩을 켜는 사람은 산문을 쓰는 사람이고, 산문에 줄 번호는 별 의미가 없다. (b)는 비용 대비 이득이 없다.

토글은 툴바 버튼 + 설정(localStorage 저장, `theme.js`가 이미 쓰는 패턴)으로 노출한다.

## 파일

- `src/renderer/editor.js` — 리스트 이어쓰기, ⌘B/⌘I, 랩 토글. 순수 함수(접두사 계산, 마커 토글)는 별도로 뽑아 단위 테스트 가능하게.
- `src/renderer/index.html` — `#source-editor`의 `white-space`, 랩 모드 클래스, 툴바 버튼
- `tests/unit/editor.test.js` — 순수 함수 테스트

## 검증

- 단위(여기가 대부분): 리스트 접두사 계산 — 불릿/번호/체크박스/중첩 들여쓰기/빈 항목 탈출/리스트 아닌 줄. 마커 토글 — 감싸기·벗기기·선택 없음.
- e2e: 소스 모드에서 `- item` 입력 → Enter → 다음 줄이 `- `로 시작하는지. 한 번 더 Enter → 접두사가 사라지는지.
- e2e: 텍스트 선택 → ⌘B → `**...**`로 감싸지는지 → 다시 ⌘B → 벗겨지는지.
- **undo 회귀 테스트**: 위 동작들 각각 수행 후 ⌘Z → 한 번에 되돌아가는지. `execCommand`를 쓰는 이유가 이것이므로 반드시 확인.
- e2e: 랩 토글 시 거터가 숨겨지는지(위 함정에 대한 직접 검증).
